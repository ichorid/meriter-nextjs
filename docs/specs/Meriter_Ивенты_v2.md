## 1\. Общая концепция

### Что такое ивент

Ивент — это **тип поста** внутри сообщества или проекта. Не проект, не отдельная сущность — пост с надстройкой. У него есть дата проведения, место, список участников и механика начисления заслуг участникам.

Ивент живёт **внутри** сообщества или проекта. Он не публикуется на Биржу. Его карточка видна всем (публичная), но RSVP доступен только участникам сообщества/проекта.

### Где ивенты существуют

- Внутри **сообщества** — в разделе «Ивенты сообщества»  
- Внутри **проекта** — в разделе «Ивенты проекта»

По аналогии с тем, как сейчас устроен раздел «Посты на бирже»: кнопка → лента → создание.

### Пользовательский путь

1. Захожу в сообщество (или проект)  
2. Нажимаю «Ивенты»  
3. Вижу ленту: предстоящие → прошедшие  
4. Нажимаю «Новый ивент» (если есть права)  
5. Заполняю: название, описание, дата, время, место  
6. Участники сообщества видят ивент, нажимают «Пойду»  
7. Ивент проходит  
8. Кто угодно из тех, кто видит список участников, может начислить заслуги конкретному участнику  
9. Ивент остаётся в ленте как архивная запись

### Примеры

- Экологическое сообщество: «Субботник в парке Горького — 15 мая»  
- Предприниматели: «Нетворкинг-встреча \#4 — 22 мая, 19:00»  
- Проект «Шелтер»: «День открытых дверей приюта — 1–2 июня»

## 2\. Ивент как тип поста

### Архитектурное решение

Ивент — это пост с `postType = 'event'`. Он хранится в той же коллекции, что и обычные посты, но имеет дополнительные поля и отличное поведение.

### Чем ивент отличается от обычного поста

| Характеристика | Обычный пост | Пост-ивент |
| :---- | :---- | :---- |
| Голосование меритами за пост | Да | Нет |
| Дата проведения | Нет | Да (обязательная) |
| Место | Нет | Да (опциональное) |
| Список участников (RSVP) | Нет | Да |
| Начисление заслуг участникам | Нет | Да (через список участников) |
| Комментарии | Да | Да (+ автокомментарии о заслугах) |
| Публикация на Биржу | Да (через «На биржу») | Нет |
| Редактирование | Ограничено | Да (дата, описание, место) |

### Что общего с обычным постом

- Текст и описание (контент поста)  
- Комментарии  
- Автор  
- Принадлежность к сообществу/проекту  
- Хранение в той же коллекции

## 3\. Создание ивента

### Кто может создать

Настраиваемый параметр сообщества/проекта:

| Настройка | Значение | Описание |
| :---- | :---- | :---- |
| `eventCreation = 'admin'` | По умолчанию | Только админ/лид сообщества или проекта |
| `eventCreation = 'members'` | Опционально | Любой участник сообщества/проекта |

### Форма создания

| Поле | Обязательное | Описание |
| :---- | :---- | :---- |
| Название | Да | «Субботник в парке Горького» |
| Описание | Да | Что за мероприятие, программа, зачем |
| Дата начала | Да | Когда ивент начинается |
| Дата окончания | Да | Для однодневного — совпадает с датой начала |
| Время | Нет | Текстовое поле: «10:00 — 16:00» |
| Место | Нет | Текстовое поле: адрес / «Онлайн» / ссылка |

### Что происходит при создании

1. Создаётся пост с `postType = 'event'` в сообществе/проекте  
2. Ивент появляется в ленте ивентов сообщества/проекта  
3. Автор автоматически добавлен в список участников (RSVP)

## 4\. RSVP: список участников

### Механика

Простая кнопка «Пойду» — нажал, попал в список. Нажал ещё раз — вышел из списка.

### Кто может нажать «Пойду»

Прежде всего, участники сообщества или проекта, к которому привязан ивент.  
Также должна быть возможность пригласить сторонних участников, по одноразовой или ограниченной по счетчику ссылке или напрямую в приложении от участника через выбор аккаунта-цели. Проработай подходящее решение.

### Что видно в списке участников

| Элемент | Описание |
| :---- | :---- |
| Аватар и имя | Участник |
| Кнопка «Начислить заслуги» | У каждого участника в списке (видна всем, кто видит пост-ивент) |
| Общее количество | «12 участников» |

### Видимость

Карточка ивента (название, дата, место, описание, количество участников) **видна всем** — в том числе не-членам сообщества, если они просматривают карточку сообщества. Но кнопка «Пойду» и полный список участников — только для участников сообщества/проекта.

## 5\. Начисление заслуг участникам ивента

### Механика

В списке участников ивента у каждого есть кнопка «Начислить заслуги». Нажать может кто угодно, кто видит этот список (= любой участник сообщества/проекта).

### Диалог начисления

При нажатии открывается диалог:

| Поле | Описание |
| :---- | :---- |
| Получатель | Предзаполнен (тот, на чью кнопку нажали) |
| Количество | Сколько заслуг передать |
| Комментарий | Обязательное поле: за что начисляются заслуги |
| Источник | Кошелёк сообщества/проекта текущего пользователя, или глобальный кошелёк |

### Куда зачисляются

Вне зависимости от того, из какого кошелька списаны — заслуги зачисляются **на кошелёк сообщества/проекта получателя**. Если передаются глобальные мериты, они конвертируются 1:1 в заслуги сообщества/проекта.

### Автокомментарий

После начисления под постом-ивентом автоматически появляется комментарий:

**Иван Петров** начислил **Марии Сидоровой** 15 заслуг: «Отлично организовала логистику субботника»

Это обычный комментарий, видимый всем участникам сообщества/проекта.

### Связь с фичей «Передача заслуг»

Начисление заслуг в ивенте — это **тот же механизм**, что и общая передача заслуг между участниками (описана в отдельном ТЗ «Передача заслуг»). Разница только в точке входа: в ивенте кнопка находится в списке RSVP, а в сообществе — в списке участников. При начислении в контексте ивента дополнительно создаётся автокомментарий под постом-ивентом.

## 6\. Временны́е параметры и статусы

### Статусы ивента

| Статус | Условие | Визуал |
| :---- | :---- | :---- |
| 📅 Предстоящий | Текущая дата \< дата начала | Обратный отсчёт: «Через 12 дней» / «Завтра» / «Сегодня» |
| 🟢 Идёт | Дата начала ≤ текущая дата ≤ дата окончания | «Идёт сейчас» |
| 📋 Прошёл | Текущая дата \> дата окончания | «Прошёл 3 дня назад» |

Статус вычисляется динамически на клиенте (не хранится в БД, не требует cron). Просто сравнение текущей даты с `eventStartDate` / `eventEndDate`.

### Многодневные ивенты

Ивент может длиться несколько дней: дата начала — 1 июня, дата окончания — 3 июня. Отображение: «1–3 июня».

## 7\. Отображение: лента ивентов

### Точка входа

В сообществе и проекте появляется кнопка/раздел **«Ивенты»** (на том же уровне навигации, что «Посты на бирже»). При нажатии открывается лента ивентов.

### Структура ленты

Лента делится на два блока:

**Предстоящие и текущие** (сверху) — сортировка: ближайшие по дате — выше.

**Прошедшие** (снизу, или по скроллу) — сортировка: недавние — выше. Визуально приглушены (например, уменьшенная яркость или плашка «Прошёл»).

### Карточка в ленте (превью)

| Элемент | Описание |
| :---- | :---- |
| Название | «Субботник в парке Горького» |
| Дата | «15 мая, 10:00 — 16:00» или «1–3 июня» |
| Статус | 📅 «Через 5 дней» / 🟢 «Идёт сейчас» / 📋 «Прошёл» |
| Место | «Парк Горького, вход со стороны Ленинского» |
| Участники | «12 человек пойдут» \+ аватарки первых нескольких |
| Кнопка «Пойду» | Прямо в карточке (для быстрого RSVP, не заходя внутрь) |

Клик по карточке → открывается полный пост-ивент.

### Кнопка «Новый ивент»

Расположена вверху ленты. Видна тем, у кого есть права на создание (настройка `eventCreation`).

## 8\. Карточка ивента

Полный вид поста-ивента при открытии.

### Верхний блок (мета-информация)

| Элемент | Описание |
| :---- | :---- |
| Название | Крупно |
| Статус | Обратный отсчёт / «Идёт» / «Прошёл» |
| Дата и время | «15 мая, 10:00 — 16:00» |
| Место | Текстом |
| Автор | Кто создал |

### Блок описания

Текст поста — описание мероприятия, программа, правила.

### Блок RSVP

| Элемент | Описание |
| :---- | :---- |
| Кнопка «Пойду» / «Не пойду» | Переключатель |
| Список участников | С аватарками и кнопкой «Начислить заслуги» у каждого |
| Счётчик | «15 участников» |

### Блок комментариев

Обычные комментарии \+ автокомментарии о начислении заслуг. Автокомментарии визуально отличаются (иконка или пометка), но являются частью общей ленты комментариев.

## 9\. Редактирование ивента

Автор (или админ) может редактировать ивент после создания.

| Что можно изменить | Ограничения |
| :---- | :---- |
| Название | Без ограничений |
| Описание | Без ограничений |
| Дата начала / окончания | Без ограничений (даже после прошедшей даты — на случай переноса) |
| Время | Без ограничений |
| Место | Без ограничений |

При изменении даты — статус пересчитывается автоматически (клиентская логика).

---

## 10\. Права доступа

| Действие | Кто может |
| :---- | :---- |
| Видеть карточку ивента | Все (публичная) |
| Видеть полный список участников | Участники сообщества/проекта |
| Нажать «Пойду» | Участники сообщества/проекта |
| Начислить заслуги | Участники сообщества/проекта (кто видит список) |
| Создать ивент | Настройка: `admin` (по умолчанию) или `members` |
| Редактировать ивент | Автор ивента или админ сообщества/проекта |
| Удалить ивент | Автор или админ |

Также нужен функционал регистрации на ивент через QR. На месте проведения администратор показывает людям QR, они сканируют, и у них открывается Meriter, регистрируя их на ивент.

## 11\. Влияние на существующий функционал

### Что меняется

| Область | Изменение |
| :---- | :---- |
| **Модель поста** | Новый `postType = 'event'` с доп. полями |
| **Внутри сообщества** | Новый раздел «Ивенты» |
| **Внутри проекта** | Новый раздел «Ивенты» |
| **Комментарии** | Автокомментарии при начислении заслуг |
| **Настройки сообщества/проекта** | Новая настройка `eventCreation` |

### Что НЕ меняется

- Логика голосования (ивент не голосуется)  
- Логика Биржи (ивент не публикуется)  
- Логика инвестирования  
- Экономика проектов (кооператив, доли)  
- Навигация верхнего уровня  
- Тикеты

## 12\. Данные для реализации

### Новые поля в модели поста (для `postType = 'event'`)

| Поле | Тип | Описание |
| :---- | :---- | :---- |
| eventStartDate | Date | Дата начала (обязательная) |
| eventEndDate | Date | Дата окончания (обязательная) |
| eventTime | string? | Время проведения (текстом) |
| eventLocation | string? | Место (текстом) |
| eventAttendees | UserId\[\] | Список участников (нажавших «Пойду») |

### Новая настройка сообщества/проекта

| Поле | Тип | Описание |
| :---- | :---- | :---- |
| eventCreation | enum: `'admin'` / `'members'` | Кто может создавать ивенты. По умолчанию `'admin'` |

### Модель автокомментария (запись о начислении заслуг в ивенте)

Автокомментарий — обычный комментарий в коллекции комментариев, но с доп. полями:

| Поле | Тип | Описание |
| :---- | :---- | :---- |
| isAutoComment | boolean | Флаг автокомментария |
| meritTransferId | TransferId? | Ссылка на запись передачи заслуг (из фичи «Передача заслуг») |

## 14\. Порядок реализации

### Зависимости

EV5 (начисление заслуг в ивенте) **зависит от** фичи «Передача заслуг» — это один механизм. Рекомендуется реализовать «Передачу заслуг» первой, затем ивенты.

### Предлагаемая последовательность

**Шаг 1**: Передача заслуг (отдельное ТЗ) — базовый механизм.

**Шаг 2**: Ивенты.

| Этап | Задачи | Оценка |
| :---- | :---- | :---- |
| Модель данных | Новые поля в посте, настройка сообщества | 0.5 дня |
| Создание \+ редактирование | Форма, валидация | 1 день |
| Лента ивентов | Раздел в сообществе/проекте, два блока, превью-карточки | 1.5 дня |
| Карточка ивента | Полный вид \+ статусы | 1 день |
| RSVP | Кнопка, список, счётчик | 0.5 дня |
| Интеграция с передачей заслуг | Кнопка в списке RSVP \+ автокомментарий | 1 день |
| Тестирование | Все сценарии | 1 день |
| **Итого** |  | **\~6–7 рабочих дней** |

Оценка при условии, что «Передача заслуг» уже реализована.
