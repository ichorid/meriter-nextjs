# PRD: Ивенты (Events)

## Цель
Дать сообществам и проектам возможность создавать мероприятия (ивенты) с датой, местом, списком участников (RSVP) и начислением заслуг участникам.

## Контекст
- **Текущее состояние**: Внутри сообществ и проектов есть посты и комментарии. Нет способа создать мероприятие с датой, собрать RSVP, отметить участников.
- **Проблема**: Сообщества организуют акции, встречи, субботники — но нет инструмента для этого на платформе. Координация уходит во внешние каналы.
- **Связанные модули**: Posts, Comments, Community members, Community/Project navigation, Merit Transfer (зависимость).
- **Зависимость**: Требует реализованную фичу «Передача заслуг» — для начисления заслуг участникам ивента.
- **Бизнес-документ**: `docs/specs/Meriter_Ивенты_v2.md`

## Требования

### Функциональные
- [ ] FR-1: Ивент — тип поста (`postType = 'event'`) с доп. полями: дата начала, дата окончания, время (текст), место (текст), список участников (RSVP).
- [ ] FR-2: Раздел «Ивенты» в навигации сообщества и проекта. Лента: предстоящие (ближайшие сверху) + прошедшие (недавние сверху, визуально приглушены).
- [ ] FR-3: Кнопка «Новый ивент» — доступна в зависимости от настройки `eventCreation` (`admin` по умолчанию / `members`).
- [ ] FR-4: Форма создания: название, описание, дата начала, дата окончания, время (опц.), место (опц.).
- [ ] FR-5: RSVP — кнопка «Пойду» (toggle). Список участников с аватарками и счётчиком. Автор автоматически в списке.
- [ ] FR-5a: **Приглашение сторонних участников**. Помимо кнопки «Пойду» (для участников сообщества/проекта), два способа пригласить людей, не являющихся участниками: (1) **инвайт-ссылка** — одноразовая или с лимитом по счётчику, генерируется админом/автором; при переходе пользователь (с аккаунтом Meriter) попадает на страницу ивента с предложением подтвердить участие; (2) **прямое приглашение** — участник ивента или админ выбирает аккаунт-цель в приложении и отправляет ему приглашение (уведомление со ссылкой на ивент).
- [ ] FR-5b: **QR-регистрация**. Админ может показать QR-код ивента (сгенерированный из инвайт-ссылки). Люди на месте проведения сканируют QR, у них открывается Meriter, и они регистрируются на ивент. По сути QR — визуальное представление инвайт-ссылки.
- [ ] FR-6: Кнопка «Пойду» — для участников сообщества/проекта. Для **сторонних** — доступ через инвайт-ссылку или прямое приглашение (FR-5a). Карточка ивента (название, дата, место, описание, количество участников) — видна всем.
- [ ] FR-7: У каждого участника в RSVP-списке — кнопка «Начислить заслуги» (механизм = передача заслуг). При начислении в контексте ивента — создаётся автокомментарий под постом-ивентом.
- [ ] FR-8: Статусы: 📅 Предстоящий / 🟢 Идёт / 📋 Прошёл — вычисляются на клиенте по дате.
- [ ] FR-9: Редактирование ивента автором или админом (все поля).
- [ ] FR-10: Голосование за пост-ивент отключено. Ивент не публикуется на Биржу.
- [ ] FR-11: Комментарии под ивентом — работают как обычно + автокомментарии о начислении заслуг.
- [ ] FR-12: Удаление ивента — доступно автору или админу.
- [ ] FR-13: Уведомление о создании ивента — участники сообщества/проекта получают уведомление «В сообществе X создан ивент Y — Z числа».

### Технические
- [ ] TR-1: Расширить модель Post — новые поля для `postType = 'event'`: eventStartDate, eventEndDate, eventTime, eventLocation, eventAttendees.
- [ ] TR-2: Новая настройка сообщества/проекта: `eventCreation` (enum: `admin` / `members`).
- [ ] TR-3: API: create event, update event, RSVP (attend / unattend), get events by community.
- [ ] TR-3a: API: `createInviteLink(eventId, options: { maxUses?: number, oneTime?: boolean })` — генерация инвайт-ссылки с токеном. `attendViaInvite(token)` — RSVP по инвайт-ссылке (доступен любому пользователю Meriter, не только участникам сообщества).
- [ ] TR-3b: API: `inviteUser(eventId, targetUserId)` — прямое приглашение пользователя; создаёт уведомление со ссылкой на ивент.
- [ ] TR-3c: Frontend: генерация QR-кода из инвайт-ссылки (клиентская библиотека QR). Кнопка «Показать QR» у админа/автора на странице ивента → отображение QR в модальном окне или фулл-скрине (удобно показывать на экране/проекторе).
- [ ] TR-4: При начислении заслуг в контексте ивента — вызов MeritTransferService + создание автокомментария.
- [ ] TR-5: Автокомментарий — обычный комментарий с `isAutoComment: true` и `meritTransferId`.

- **Новые/изменяемые сервисы**: расширить PublicationService (или аналог) для создания/обновления event-постов; RSVP-логика; генерация инвайт-ссылок; RSVP по токену; прямое приглашение (уведомление)

## Критические тесты
- [ ] **Создание ивента**: postType='event', eventStartDate/eventEndDate заполнены, автор в eventAttendees → запись в БД корректна.
- [ ] **RSVP attend/unattend**: attend → userId в eventAttendees; повторный attend → без дубликата; unattend → userId удалён.
- [ ] **Голосование заблокировано**: попытка проголосовать за postType='event' → ошибка (не стандартная логика голосования).
- [ ] **Forward заблокирован**: попытка переслать/опубликовать event на биржу → ошибка.
- [ ] **Инвайт-ссылка (oneTime=true)**: создать, использовать → usedCount=1, повторное использование → ошибка.
- [ ] **Инвайт-ссылка (maxUses=3)**: использовать 3 раза → ok; 4-й раз → ошибка.
- [ ] **attendViaInvite**: пользователь НЕ участник сообщества + валидный токен → попадает в eventAttendees (НЕ в participants сообщества).
- [ ] **Начисление заслуг в ивенте**: вызов MeritTransferService с eventPostId → MeritTransfer запись с eventPostId, автокомментарий создан под постом-ивентом.
- [ ] **Автокомментарий**: isAutoComment=true, meritTransferId заполнен, текст содержит имя отправителя, получателя, сумму.
- [ ] **Существующие посты**: создание обычного поста (postType=undefined) в обычном сообществе → без изменений в поведении.

## Заметки
- Ивент — это пост, а не проект. У него нет тикетов, фаз, кооперативной экономики.
- Статусы ивента вычисляются на клиенте (сравнение текущей даты с eventStartDate/eventEndDate), не хранятся в БД.
- Многодневные ивенты поддерживаются (дата начала ≠ дата окончания).
- Инвайт-ссылки и QR — единый механизм. QR — визуальное представление инвайт-ссылки. Генерация QR — на клиенте (библиотека `qrcode.react` или аналог), бэкенд не генерирует изображения.
- Приглашённые по инвайт-ссылке пользователи попадают **только в список участников ивента** (eventAttendees), а НЕ в участники сообщества/проекта. Это важное разграничение.
- **Краевой случай (начисление заслуг приглашённым)**: приглашённые по инвайт-ссылке не являются участниками сообщества/проекта. Им можно начислить заслуги только из глобального кошелька → на их глобальный кошелёк. Локальная передача для них невозможна.
