## 1. Цель

Провести **полноценный визуальный и UX-редизайн**:

1. Экран **просмотра профиля** в двух режимах: **свой** (`/meriter/profile`) и **чужой** (`/meriter/users/[userId]`).
2. **Адаптив**: отдельные решения для **mobile** и **desktop**.
3. **Глобальная навигация**: согласованное обновление **нижней панели** и **левого сайдбара** (и при необходимости **верхней зоны** приложения), чтобы визуально и информационно совпадать с новым профилем.

Редизайн **не меняет бизнес-правил** начисления меритов, прав доступа и контрактов API, кроме случаев, где это явно перечислено в разделе «Документация». Новые продуктовые сущности (верификация личности, «уровни», web3-ID и т.д.) **в scope не входят**.

---

## 2. Контекст и проблема

- **Сейчас**: функционал профиля и навигации покрывает домен (глобальные мериты, история, активность, сообщества/проекты, избранное, инвестиции, приглашения и перевод меритов на чужом профиле), но визуальная иерархия и плотность интерфейса уступают целевому образу платформы.
- **Референс визуала**: макеты Stitch (Tailwind, токены поверхности/primary, шрифт Manrope, Material Symbols, bento-сетка, «стеклянные» панели). HTML-референсы согласованы с командой; имплементация — **в коде Next.js**, а не вставка сырого HTML.
- **Проблема**: пользователю сложнее сканировать профиль; навигация разнесена по паттернам (часть в сайдбаре, часть в герое); внешний вид не отражает желаемый «продуктовый» уровень polish.

| **Подстраницы профиля**  | Маршруты под `/meriter/profile/*` (публикации, комментарии, опросы, избранное, инвестиции, merit-transfers и т.д.): **минимально** — единый визуальный язык (отступы, типографика, шапки); при отдельном спринте можно вынести в follow-up, но PRD закладывает **единый стиль**. |

| **Наблюдаемость**        | Логирование ключевых действий пользователя в профиле и навигации (раздел 9).                                                                                                                                                                                                     |
| **Документация**         | Обновление business-rules и frontend-rules после стабилизации IA (раздел 10).                                                                                                                                                                                                    |


### 3.2 Вне scope (явный запрет без отдельного PRD)

- Верификация личности, бейджи «verified», «Meriter Trust», совет сообществ.
- Digital ID, web3-адреса, фиксированные поля «Telegram» / соцсети, если их нет в модели пользователя.
- Метрики «за неделю», «+N%», «уровень», gamification-бейджи без бэкенда.
- Выдуманные числа участников, инвестированных сумм и статусов проектов — только данные из API.
- Изменение правил `meritTransfer`, RSVP, кошельков, квот и т.п., кроме отображения.

- `web/src/components/organisms/VerticalSidebar/VerticalSidebar.tsx`
- `web/src/components/organisms/BottomNavigation.tsx`
- `web/src/components/templates/AdaptiveLayout/*` (при смене chrome)

- `web/src/components/organisms/CommunityCard.tsx` (визуал карточек в профиле и сайдбаре)
- i18n: `web/messages/*` (или актуальный путь к переводам)

## 4. Принципы (зафиксированные решения)

1. **Нулевой функциональный регресс**: ни один существующий контроль не удаляется; новые контролы не добавляются без продукта и API.
2. **Перенос, а не выброс**: элементы можно переносить между hero / sidebar / top bar, но каждый **сохраняет маршрут или обработчик** из текущей версии.
3. **Данные только из API**: любые числа и подписи на карточках сообществ/проектов — из текущих хуков и моделей.
4. **Два режима профиля**: явное различие UI для «свой» / «чужой» только там, где оно уже есть в логике (редактирование, настройки, приглашение, перевод меритов).
5. **i18n**: пользовательские строки — через `next-intl`; не копировать застрявшие в макете термины, противоречащие глоссарию (например «Заслуги» vs «мериты») без согласования с `docs/business-docs-ru` / владельцем копирайта.
6. **Доступность**: контраст, фокус-кольца, `aria-label` на иконках, размеры тач-таргетов ≥ 44px где возможно, сохранение семантики кнопок/ссылок.

| FR-P-06 | Секция **активности** (`ProfileContentCards`): для **своего** профиля **пять** карточек — публикации, комментарии, опросы, избранное, инвестиции (маршруты и хуки без изменения контрактов). | FR-P-07 | Секции **администрируемые сообщества**, **участник сообществ**, **администрируемые проекты**, **участник проектов** + пустые состояния + `**CommunityCard`** с **кошельком/квотой** на своём профиле и без них на чужом.                                                                                                                                                                                                                                                     |
| FR-P-08 | Кнопка **«Создать сообщество»** → `/meriter/communities/create` остаётся.                                                                                                                                                                                                                                                                                                                                                                                                    |
| FR-P-09 | Режим **редактирования** своего профиля (`ProfileEditForm`) сохраняется.                                                                                                                                                                                                                                                                                                                                                                                                     |
| FR-P-10 | На **чужом** профиле при `viewingOtherProfile`: **Invite to team** + `**MeritTransferButton`** при непустом `profileMeritTransfer`; диалог приглашения без изменения API.                                                                                                                                                                                                                                                                                                    |
| FR-P-11 | Для своего профиля в герое: **Edit**, **Settings** (`/meriter/settings`), **Share** — те же действия, что в `ProfileHero`.                                                                                                                                                                                                                                                                                                                                                   |

## 7. Дизайн-система и визуал

- База: тёмная тема, токены уровня surface / primary / outline (как в референсе Stitch или маппинг на существующие `base-`* / brand — решение в задаче на имплементацию с **одной** согласованной схемой).
- Типографика: **Manrope** (или текущий primary font, если отклонение от референса зафиксировано в progress).
- Иконки: согласовать **один** стиль на всём scope (lucide vs Material Symbols); смешение минимизировать.
- Карточки: bento, мягкие границы, hover/active для интерактивных зон.
- **Запрещено** визуально показывать данные, которых нет в модели (см. §3.2).

## 9. Логирование действий и наблюдаемость

Цель: измерять использование нового профиля и навигации, не нарушая приватность (без PII в событиях: не логировать email, телефон, содержимое bio в payload).

### 9.1 Клиент (web)

1. **Ввести тонкий слой** `profileUiTelemetry` (или общий `uiTelemetry`):
  - обёртка над `Sentry.addBreadcrumb` (всегда при включённом Sentry);
  - опционально: `window`/`fetch` к endpoint аналитики при `config.features.analytics === true` (см. `web/src/config/index.ts`).
2. **События (минимальный набор v1)**:

  | Событие                                       | Когда                                          | Полезная нагрузка (без PII)                                                                                                                      |
  | --------------------------------------------- | ---------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
  | `profile_view_self`                           | Успешный mount `/meriter/profile`              | `{ layout: 'sm' | 'lg' }`                                                                                                                        |
  | `profile_view_other`                          | Mount `/meriter/users/:id`                     | `{ targetUserId }` (id допустим как внутренний идентификатор сущности)                                                                           |
  | `profile_activity_card_click`                 | Клик по карточке активности                    | `{ card: 'publications'|'comments'|'polls'|'favorites'|'investments', scope: 'self'|'other' }`                                                   |
  | `profile_merit_history_open`                  | Клик по ссылке полной истории меритов          | `{ scope: 'self'|'other' }`                                                                                                                      |
  | `profile_share`                               | Успешный share                                 | `{}`                                                                                                                                             |
  | `profile_edit_open` / `profile_settings_open` | Клик Edit / Settings                           | `{}`                                                                                                                                             |
  | `nav_primary_click`                           | Клик по пункту первичной навигации             | `{ item: 'future_visions'|'marathon'|'projects'|'notifications'|'favorites'|'profile'|'support'|'about', surface: 'sidebar'|'bottom'|'topbar' }` |
  | `nav_create_click`                            | Клик «Создать» (любая поверхность)             | `{ surface: 'sidebar'|'fab' }`                                                                                                                   |
  | `profile_invite_open`                         | Открытие приглашения в команду (чужой профиль) | `{}`                                                                                                                                             |
  | `profile_merit_transfer_open`                 | Открытие перевода меритов с профиля            | `{}`                                                                                                                                             |

3. **Согласование с Sentry**: не поднимать уровень шума — `category: 'ui.profile'`, `level: 'info'` для breadcrumbs.

### 9.2 Сервер (api)

- **В v1 не требуется** новая коллекция audit для кликов по UI.
- Если позже понадобится централизованный аудит: отдельный PRD; до тех пор достаточно клиентских breadcrumbs + существующих server logs на мутациях.

### 9.3 Конфиденциальность

- Не логировать: содержимое полей профиля, контакты, токены, query-параметры с секретами.
- Соблюдать политику cookies/аналитики продукта (при появлении баннера согласия — учесть в будущем).

## 11. План внедрения (рекомендуемые этапы)

1. **Токены и оболочка**: тема, отступы `AdaptiveLayout`, опциональный top bar без изменения маршрутов.
2. **Навигация**: `VerticalSidebar` + `BottomNavigation` под новый визуал + телеметрия.
3. **Профиль**: `ProfileHero` + merits strip + stats + content cards.
4. **Списки сообществ**: рескин `CommunityCard` в контексте профиля/сайдбара.
5. **Чужой профиль**: выравнивание с теми же компонентами + блок действий.
6. **Подстраницы профиля**: единый стиль шапок и контента.
7. **Полировка a11y, i18n, тесты, bump версии web.**

Этапы можно объединять в PR, но **FR по инвариантам** должны выполняться до объявления фичи «готово».

## 12. Критерии приёмки (QA)

- Свой профиль: все элементы из §5.1 присутствуют; пять карточек активности; `ProfileStats` виден при ненулевых данных API.
- Чужой профиль: три карточки; приглашение и перевод меритов — как до редизайна (по условиям данных).
- Навигация: чеклист §5.2; избранное и уведомления доступны с desktop; long-press профиля и CreateMenu работают.
- Нет UI элементов из §3.2.
- Линт, тесты, сборка — успешно.
- Breadcrumbs/события телеметрии срабатывают в dev (проверка в Sentry debug или console при отключенном Sentry — не ломать production).
- Регрессия основных маршрутов из профиля вложенных страниц.
