# PRD: Редизайн страницы проекта

## Цель
Привести страницу проекта к профессиональному, интуитивно понятному дизайну — сохраняя ВСЕ существующие контролы и функционал. Текущий вид выглядит как черновик: элементы разбросаны без визуальной иерархии, пустые состояния не оформлены, нет логичной группировки.

## Контекст
- **Текущее состояние**: Страница проекта содержит все нужные контролы (статус, описание, кошелёк, задачи, обсуждения, доли, публикация на бирже, управление), но выглядит как набросок: плоская структура, нет визуальных разделителей, кнопки действий свалены в одну строку внизу
- **Проблема**: Неуклюжий UX — пользователь не понимает иерархию действий, не видит структуры, кнопки управления проектом визуально не разделены от рабочих действий
- **Связанные модули**: Страница сообщества (как reference — она уже ближе к финальному виду), компоненты задач, кошелёк, биржа

## Принципы дизайна

### 1. Визуальная иерархия (сверху вниз)
Страница должна читаться естественно: «что это» → «как дела» → «что делать» → «настройки».

### 2. Консистентность с остальной платформой
Использовать те же паттерны, что на странице сообщества: cover-картинка + аватар, карточки с мягкими бордерами, тёмная тема с полупрозрачными разделителями.

### 3. Группировка по смыслу
Контролы сгруппированы по назначению, а не свалены в одну полосу. Деструктивные действия визуально отделены от рабочих.

### 4. Empty states
Каждый пустой блок (задачи, участники, обсуждения) должен иметь осмысленное пустое состояние с CTA.

## Структура страницы (Layout)

### Зона 1 — Hero / Идентификация проекта
**Элементы:**
- Cover-картинка проекта (или дефолтный градиент по `typeTag`)
- Аватар / иконка проекта (поверх cover, как на странице сообщества)
- Название проекта
- Бейдж статуса: `Активен` / `Завершён` / `Архив` — цветовой бейдж рядом с названием
- Описание проекта (1-2 строки, с раскрытием по клику)
- Ссылка на родительское сообщество (кликабельная, со стилем `text-blue-400 hover:underline`)
- Теги / категории (если есть)

**Дизайн-решение:** Повторить паттерн hero-секции со страницы сообщества. Cover — 200px высота, аватар — 72px, наложен на нижнюю границу cover. Кнопки управления (удалить, настройки, участники) — в правом верхнем углу cover как icon-кнопки с полупрозрачным фоном (как на странице сообщества).

### Зона 2 — Dashboard-карточки (ключевые метрики)
**Горизонтальный ряд из 2-3 карточек:**

**Карточка «Кошелёк проекта»:**
- Иконка кошелька
- Баланс: `0 merits` (крупный шрифт)
- Разбивка: `Прочие: 100% —` (мелким текстом)
- Кнопка «Пополнить» — secondary button

**Карточка «Команда»:**
- Количество участников (крупная цифра)
- Аватары первых 3-4 участников (stack)
- Ссылка «Все →»

**Карточка «Доли в проекте»:**
- Визуальное отображение долей (mini progress-bar или donut)
- Текстовое описание текущего распределения
- Кнопка «Обновить доли» (если есть права)

**Дизайн-решение:** Карточки — `bg-white/5 border border-white/10 rounded-xl p-4`. На мобильном — вертикальный стек. На десктопе — ряд `grid grid-cols-3 gap-4`.

### Зона 3 — Рабочая область (Задачи + Обсуждения)
**Tabs: `Задачи` | `Обсуждения`** (как сейчас, но стилизованные)

**Под табами:**
- Фильтр задач: dropdown `Все / Открытые / В работе / Завершённые`
- Кнопки создания: «Создать задачу» (primary), «Создать открытую задачу» (secondary/outline)

**Список задач:**
- Каждая задача — карточка с: название, статус-бейдж, аватар исполнителя (или `?` для открытых), дата
- Empty state: иконка + текст «Пока нет задач. Создайте первую задачу, чтобы начать работу» + CTA-кнопка

**Вкладка «Обсуждения»:**
- Список постов (используется существующий компонент ленты)
- Кнопка «Создать пост»

**Дизайн-решение:** Табы — underline-стиль (`border-b-2 border-blue-500`), активный таб — белый текст, неактивный — `text-gray-400`. Контент табов — внутри карточки `bg-white/5 rounded-xl`.

### Зона 4 — Действия проекта (Actionbar)
**Две визуальные группы, разделённые:**

**Группа A — Рабочие действия (публичные):**
- 🚀 «Опубликовать на Бирже» — primary, акцентная кнопка (зелёная, как сейчас `bg-green-600`)
- Видна только если есть что публиковать

**Группа B — Административные действия:**
Внутри карточки `bg-white/5 rounded-xl p-4` с заголовком «Управление проектом»:
- «Передать админку проекта» — ghost/text button
- «Обновить доли» — ghost/text button
- «Выйти из проекта» — ghost/text button, цвет `text-gray-400`

**Отдельно, с визуальным разрывом:**
- «Закрыть проект» — деструктивная кнопка `bg-red-600/10 text-red-400 border border-red-600/20`, с иконкой ⚠️

**Дизайн-решение:** НЕ сваливать все кнопки в одну горизонтальную строку (как сейчас). Группировать по смыслу и уровню опасности. Деструктивные действия — максимально отделены визуально.

## Детальные требования к компонентам

### FR-1: Hero-секция
- [ ] Cover-изображение проекта (fallback — градиент на основе цвета сообщества или `typeTag`)
- [ ] Аватар проекта поверх cover (patern: страница сообщества)
- [ ] Название + бейдж статуса в одну строку
- [ ] Описание — truncated, expand on click
- [ ] Ссылка на родительское сообщество
- [ ] Admin-иконки в правом верхнем углу cover (удалить, настройки, участники) — показывать только админам

### FR-2: Dashboard-метрики
- [ ] Карточка кошелька с балансом и кнопкой пополнения
- [ ] Карточка команды с количеством и аватарами
- [ ] Карточка долей с визуализацией (если доли настроены)
- [ ] На мобильном — вертикальный стек, на десктопе — grid

### FR-3: Рабочая область с табами
- [ ] Tab-переключатель: Задачи / Обсуждения
- [ ] Toolbar для задач: фильтр + кнопки создания
- [ ] Empty states для обеих вкладок с CTA
- [ ] Сохранять выбранный таб при навигации (URL param или state)

### FR-4: Список задач (refactor)
- [ ] Каждая задача — карточка (не строка таблицы)
- [ ] Статус-бейдж: `Открытая` (жёлтый), `В работе` (синий), `Завершена` (зелёный)
- [ ] Аватар бенефициара или placeholder для нейтральных тикетов
- [ ] Hover-эффект на карточке

### FR-5: Action-группы
- [ ] «Опубликовать на Бирже» — отдельная акцентная кнопка, видимая всем кто имеет права
- [ ] Блок «Управление проектом» — группа ghost-кнопок внутри карточки
- [ ] «Закрыть проект» — визуально отделена, destructive style
- [ ] Все admin-only кнопки скрыты для обычных участников

## Технические требования

### TR-1: Компонентная структура
Создать или рефакторить:
- `ProjectHero` — hero-секция (cover + avatar + info)
- `ProjectDashboard` — ряд метрик-карточек
- `ProjectWorkArea` — табы + контент (задачи / обсуждения)
- `ProjectActions` — группированные действия
- `ProjectTaskCard` — карточка задачи
- `ProjectEmptyState` — переиспользуемый empty state

### TR-2: Стилизация
- Использовать существующий Tailwind + cn() utility
- Тёмная тема: `bg-gray-900` основной фон, карточки `bg-white/5 border border-white/10`
- Акценты: зелёный для CTA (`bg-green-600`), красный для деструктивных (`bg-red-600/10 text-red-400`)
- Border-radius: `rounded-xl` для карточек, `rounded-lg` для кнопок
- Анимации: `transition-colors duration-200` на interactive элементах

### TR-3: Responsive
- Mobile-first: все карточки — вертикальный стек
- Desktop (≥768px): dashboard-карточки — grid, hero шире
- Кнопки на мобильном — full-width

## Ограничения
- [ ] Не ломать: существующий функционал проекта (задачи, кошелёк, доли, биржа, управление)
- [ ] Не ломать: пермишены (admin-only кнопки должны остаться admin-only)
- [ ] Не ломать: роутинг и API-вызовы
- [ ] Совместимость: с текущей тёмной темой приложения
- [ ] Все существующие кнопки и действия должны быть сохранены — только перегруппированы
