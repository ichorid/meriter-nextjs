# Platform wipe and demo seed

Superadmin-only tools in **About → Platform settings** (gear on the About page):

1. **Wipe platform data** — deletes **all** users (including superadmins), then recreates a single bootstrap superadmin (`PLATFORM_WIPE_SUPERADMIN` in `api/apps/meriter/src/domain/common/constants/platform-bootstrap.constants.ts`: email `dmitrsosnin@gmail.com`, display name Дмитрий Соснин, bio «Команда Меритер: технический администратор»). Deletes local communities and all dependent collections (publications, votes, wallets, etc.), clears feeds on the four priority hubs + global merit community, applies **`api/apps/meriter/seed-data/dev-platform-snapshot.json`** to `platform_settings` and priority hubs + global community (if the file is missing or invalid, code falls back to `PUBLIC_PLATFORM_SETTINGS_BOOTSTRAP` / `PRIORITY_HUB_BOOTSTRAP`), and clears the demo-seed marker. **Keeps** `categories` and About content (`about_categories` / `about_articles`). The UI also requires typing `WIPE`, a short countdown, and the extra operator password checked server-side (`PLATFORM_WIPE_EXTRA_PASSWORD` in `api/apps/meriter/src/domain/common/constants/platform-dev.constants.ts`, default `1243`).

   **Replacing the snapshot:** copy settings from your dev Mongo (or edit JSON) so `dev-platform-snapshot.json` matches the desired hub/platform defaults; commit the file for repeatable deploys. Runtime defaults for **new** local teams/projects still come from `CommunityDefaultsService` in code unless you extend that separately.

2. **Fill demo data** — reads `api/apps/meriter/seed-data/meriter-demo-projects.tsv` (UTF-8 tab-separated; keep in sync with `docs/Meriter_Demo_Projects.tsv`). Expects **41** rows. Enables Decree 809 value tags, creates **four** email superadmins (Владислав, Руслан, Ольга, Софья — addresses in `platform-demo-seed.service.ts`), and creates **41** `project` posts in **Биржа** (`marathon-of-good`) with titles, bodies, image URLs, and Decree 809 tags from the sheet. Applies weighted votes from those four plus Дмитрий so each post’s rating is in **10…1000**; **10** posts include investing (pool + withdrawals) and TTL. Final global-wallet balances are normalized to fixed demo targets (≥ 100, all distinct). Idempotent: a second run fails until you wipe (unless using `force` on the API, not recommended on shared environments).

**Recommended order:** wipe → seed.

**Production:** There is no environment gate on the server; the UI warns that wipe is destructive. Optional future env `MERITER_DISABLE_PLATFORM_WIPE` is reserved for a hard-stop but is not implemented yet.
