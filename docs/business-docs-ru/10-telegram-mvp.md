# 10. Meriter Telegram MVP

**Заслуги в Telegram-группе + обрезанная веб-платформа**

**Версия:** 1.0 (согласовано с бизнесом)  
**Фаза 1 (текущая реализация):** автономный Telegram-бот — вся меритономика в группе без веб-MVP.  
**Фаза 2:** усечённый Meriter (одно сообщество, проекты во вкладке, Telegram Login, зеркало постов из веба).

---

## 1. Резюме для руководства

### Проблема

Команды и сообщества уже живут в Telegram: обсуждают идеи, помогают друг другу, координируют проекты. При этом нет простого способа **измерить вклад**, **поблагодарить заслугами** и **принимать решения на основе накопленного доверия** — не выходя из привычного чата.

### Решение

**Meriter Telegram MVP** — связка из:

1. **Telegram-бота** — основной интерфейс для ежедневной работы с заслугами: баланс, голосование, переводы, уведомления.
2. **Веб-платформы Meriter (обрезанной)** — для сложного контента и управления: посты с оформлением, опросы, совместные документы, события, проекты, история, настройки.

Одна Telegram-группа = одно сообщество на платформе. Участник группы автоматически становится участником сообщества. Баланс квоты и кошелька **один и тот же** в чате и в вебе.

### Ценность

- Меритономика **там, где уже есть команда** — не нужно уводить людей на отдельный сервис для простых действий.
- Веб остаётся для **серьёзного контента**, но не обязателен для базовых операций.
- На базе уже существующей платформы Meriter — не «с нуля», а целенаправленное упрощение и усиление Telegram-сценария.

### Границы MVP

**Входит:** одно (или пилотное) сообщество; бот; веб с функциями внутри сообщества; проекты **внутри вкладки сообщества**; регистрация только через Telegram.

**Не входит:** глобальные ленты (Образы будущего, Биржа, общий каталог проектов), социнвестиции, tappalka, пересылка между сообществами, создание новых сообществ пользователями, вход через email/SMS.

### Масштабирование

Пилот — **одна группа**. Продукт проектируется так, чтобы позже **подключать другие группы** (каждая — своё сообщество), без переписывания концепции.

## 2. Для кого этот продукт

### Организатор / админ группы

- Подключает бота к Telegram-группе.
- Настраивает правила заслуг (квота, стоимость постов, модерация).
- Управляет ролями и видит историю активности.
- Может работать и в чате, и в вебе.

### Участник группы

- Появляется в Meriter **автоматически**, когда его добавили в Telegram-группу.
- Оценивает посты реакциями прямо в чате.
- Смотрит баланс (**с долей % от общего пула**), переводит заслуги, получает **единственное** push-уведомление — когда **ему перевели** заслуги.
- При необходимости переходит в веб — создать опрос, документ, событие, проект.

### Лид сообщества (Meriter)

- Синхронизируется с администраторами Telegram-группы (предложение: автоматически).
- Полный доступ к настройкам сообщества в вебе.
- Модерация контента — если включена в настройках.

## 3. Два интерфейса: чат и веб

У продукта два «экрана», но **одна экономика заслуг**.

### Telegram-бот (основной)

**Назначение:** ежедневные действия без выхода из мессенджера.

**Что можно делать:**

- Смотреть баланс кошелька, остаток квоты и **свою долю %** от суммы заслуг активных участников (`/баланс`).
- Смотреть рейтинг участников с балансами и **%** (`/участники`).
- Смотреть **общий фонд** сообщества/проекта и личную долю в нём, если есть `CommunityWallet` (`/фонд`).
- Голосовать заслугами за посты (реакции 👍/❤️/🤡 и ответы `+N текст` / `-N текст` на якорное сообщение).
- Переводить заслуги другим участникам (**только в личку**, без сообщений в группе).
- Получать **одно** уведомление в личку: «вам перевели заслуги» (не начисления за голоса, не ошибки).
- Переходить по ссылке в веб (в **внешнем браузере** — см. раздел 10).
- Создавать простые посты (если включён режим хэштегов — см. раздел 7).

### Веб-платформа (дополнительный, «полный»)

**Назначение:** контент и управление, которые неудобно делать в чате.

**Что можно делать:**

- Лента постов сообщества.
- Создание и редактирование постов, опросов, совместных документов, событий.
- Работа над **проектами** (вкладка внутри сообщества).
- Голосование с комментарием (как в полном Meriter).
- Прямые переводы заслуг.
- История транзакций (личная и по сообществу).
- Настройки сообщества для админов.

### Как связаны чат и веб (словами)

```
Telegram-группа
    │
    ├── Бот ←→ Заслуги, голоса, переводы (общая база Meriter)
    │
    └── Ссылка «Открыть Meriter» → Веб (только это сообщество)
              │
              └── Пост / опрос / документ / событие / проект
                        │
                        └── Важные публикации дублируются в чат сообщением бота
```

## 4. Подключение группы (онбординг)

### Шаг 1. Добавить бота в группу

Админ Telegram-группы добавляет бота Meriter и выдаёт ему права:

- Читать сообщения группы (для постов по хэштегу, ответов, синхронизации участников).
- Видеть список участников (для автоматического членства).
- Отправлять сообщения (публикации, ответы, подсказки).
- Работать с реакциями (если API Telegram позволяет боту обрабатывать реакции участников).

Точный список прав уточняется на этапе разработки с учётом ограничений Telegram API.

### Шаг 2. Первичная настройка

Админ проходит короткую настройку — в вебе или в диалоге с ботом:

- Название и описание сообщества.
- Квота: включена ли, сколько заслуг в день.
- Стоимость создания поста (по умолчанию **бесплатно**).
- Нужна ли модерация постов перед публикацией (настраивается, см. раздел 11).
- Приветственные заслуги новым участникам (если предусмотрено).

### Шаг 3. Связка «чат ↔ сообщество»

Система создаёт **одно сообщество Meriter**, жёстко привязанное к этой Telegram-группе. Другие группы в этом MVP не видны пользователю.

### Шаг 4. Приветствие в чате

Бот публикует сообщение с правилами:

- как ставить реакции;
- как посмотреть баланс;
- как перейти в веб;
- к кому обращаться по вопросам.

### Типы групп

Поддерживаются **и приватные, и публичные** Telegram-группы. Логика заслуг одинакова; отличия — только в способе набора участников (invite-link vs открытая группа).

## 5. Участники и аккаунты

### Принцип «чат первичен»

**Появился человек в Telegram-группе → появился в сообществе Meriter.**

- Аккаунт создаётся автоматически по Telegram ID.
- Отдельная регистрация (email, пароль) **не нужна**.
- Вход в веб — **только через Telegram** (авторизация при переходе по ссылке).

### Первый визит

1. Участника добавили в группу.
2. Бот (или система при первом обращении) регистрирует его в Meriter.
3. Начисляются приветственные заслуги / включается дневная квота — по настройкам сообщества.
4. Человек может сразу ставить реакции и пользоваться командами бота.

### Выход из Telegram-группы — решение на согласование

Ниже три варианта. **Рекомендация команды разработки — вариант 1.**

---

**Вариант 1. Заморозка (рекомендуем)**

Человек вышел или его удалили из группы:

- в Meriter статус «заморожен»;
- **нельзя** тратить и получать заслуги через бота;
- веб — только просмотр истории (или доступ закрыт — на выбор бизнеса);
- при **повторном вступлении** в группу — автоматическая разморозка, баланс сохранён.

Плюсы: сохраняется история; соответствует принципу «чат первичен»; нет потери данных при временном выходе.

Минусы: нужно явно описать пользователю, почему бот перестал отвечать на траты.

---

**Вариант 2. Членство не зависит от чата**

Выход из Telegram **не влияет** на Meriter: человек остаётся полноправным участником, может заходить в веб и тратить заслуги.

Плюсы: простая логика.

Минусы: нарушает идею «группа = сообщество»; возможны «призраки» вне чата.

---

**Вариант 3. Полное исключение**

При выходе из группы — удаление из сообщества Meriter (как kick из команды).

Плюсы: жёсткая синхронизация.

Минусы: споры при временном бане; потеря доступа к истории; сложнее вернуть человека «как было».

---

**Для согласования:** отметьте выбранный вариант (1 / 2 / 3) или предложите гибрид.

## 6. Заслуги: квота, кошелёк, правила

### Терминология в продукте

В интерфейсе используется слово **«Заслуги»** — не «мериты», не «кредиты», не «баллы».

### Квота

- Ежедневный «бесплатный» запас заслуг на голосования и часть действий.
- **Сгорает** в конце дня: не потратил — пропало.
- Размер и вкл/выкл настраивает **админ сообщества**.
- При голосовании «в плюс» система **сначала** тратит квоту, **потом** кошелёк.

**Пример.** Квота 5 заслуг в день. Участник поставил 👍 три раза — осталось 2 в квоте. На следующий день квота снова 5 (если админ не менял настройки).

### Кошелёк

- Накопленные заслуги — **не сгорают**.
- Пополняется, когда другие голосуют за ваши посты, при переводах, при начислениях админом.
- Списывается при голосовании «в минус», при переводах, при создании платного контента, при голосовании за **свои** посты.

### Основные правила (как в Meriter)

**Голос «в плюс» (начислить автору):**
- сначала квота, затем кошелёк.

**Голос «в минус» (снять с автора):**
- только с кошелька голосующего.

**Голос за свой пост:**
- только с кошелька, квота **не** используется.

**Комиссия за создание поста / опроса:**
- настраивается админом сообщества;
- **по умолчанию — бесплатно** (0 заслуг).

### Единый баланс

Цифры в боте и в вебе **всегда совпадают**. Нет отдельного «чатового счёта».

## 7. Голосование в Telegram-чате

### Реакции (быстрые голоса)

| Реакция | Действие |
|---------|----------|
| 👍 (палец вверх) | **+1 заслуга** автору поста |
| ❤️ (сердечко) | Бот пишет **в личку**: «Сколько заслуг начислить?» → участник вводит число → подтверждение → списание и начисление |
| 🤡 (или другая «минус»-реакция) | Бот спрашивает в личку **сколько снять** → только с **кошелька** голосующего |

Если заслуг не хватает — бот сообщает об ошибке, голос **не** проводится.

### Комментарий с текстом (ответ на сообщение)

Помимо реакций, участник может **ответить (reply)** на сообщение-пост и указать заслуги с текстом.

**Пример диалога:**

```
Участник (reply на пост бота): +3 Отличная идея, поддерживаю!
Бот (в личку или в треде): Подтвердить начисление 3 заслуг автору? [Да] [Нет]
```

Для минуса — аналогично, с списанием только с кошелька.

Формат команд уточняется в UX-проработке; принцип: **reply + сумма + текст** обрабатывается ботом как полноценный голос с комментарием.

### За что можно голосовать — решение на согласование

---

**Вариант A. Зеркало постов (рекомендуем для v1)**

Голосовать можно только за **сообщения бота**, которые представляют пост в Meriter (создан в вебе, через команду боту или по согласованному правилу).

- Обычная переписка в чате **не** оценивается.
- Каждый пост в Meriter → сообщение бота в группе → на него реакции.

Плюсы: порядок, предсказуемость, проще модерация.

Минусы: «живая» переписка без явного поста не получает заслуги.

---

**Вариант B. Посты по хэштегу**

Сообщение участника с **согласованным хэштегом** (например `#заслуга`) считается постом. Реакции на **это** сообщение = голоса (если панель кнопок выключена).

Плюсы: контент рождается прямо в чате, привычно для ранних версий Meriter.

Минусы: нужен контроль формата; риск шума; бот должен видеть все сообщения.

---

**Вариант C. Любое сообщение**

Первая реакция на **любое** сообщение в группе «материализует» пост в Meriter.

Плюсы: максимальная свобода.

Минусы: хаос, флуд, сложная модерация. **Не рекомендуем для пилота.**

---

**Рекомендация:** **A обязательно**; **B опционально** как настройка сообщества («разрешить посты по хэштегу»). Вариант C — вне MVP или отдельная фаза.

**Для согласования:** A / A+B / A+B+C.

---

### Ограничения и краевые случаи

- **Self-vote:** технически возможен только с кошелька (см. раздел 6).
- **Отмена голоса:** в MVP может быть недоступна или доступна только в вебе — **уточнить на согласовании**.
- **Один пользователь — один голос на пост** или несколько — наследуется от правил Meriter для сообщества; в документе для бизнеса: «как в настройках сообщества».

## 8. Баланс, переводы и команды бота

### Просмотр баланса

Участник вызывает команду или кнопку (точные имена — на этапе UX):

**Пример ответа бота:**

```
Ваши заслуги в «Команда Альфа»

Кошелёк: 12
Квота сегодня: 3 из 5

Подробная история — в Meriter: [Открыть]
```

### Перевод другому участнику

**Пример:**

```
Участник: /перевод @ivan 5
Бот: Перевести 5 заслуг пользователю Иван? [Подтвердить] [Отмена]
Бот (после подтверждения): Готово. Ваш баланс: 7
```

Переводы — **только внутри одного сообщества** (одной группы). Правила доступности перевода — как в Meriter (роли, настройки).

### История транзакций

В чате — только краткая сводка. **Полная история** — в вебе (личная и агрегированная по сообществу для админов).

## 9. Веб-платформа MVP

### Что видит участник

После перехода по ссылке из бота (см. раздел 10) пользователь попадает в **единственное своё сообщество**. Нет глобальной навигации по «Образам будущего», «Бирже», общему каталогу проектов.

**Доступные разделы:**

1. **Лента** — посты сообщества, голосование, комментарии.
2. **Создание** — пост, опрос, документ, событие (по правам).
3. **Опросы** — создание, голосование, результаты.
4. **Документы** — совместная работа, варианты, голосование за варианты.
5. **События** — полный функционал: календарь, RSVP, приглашения (см. ниже).
6. **Проекты** — вкладка **внутри сообщества** (см. раздел 9.1).
7. **Переводы** — прямые переводы заслуг между участниками.
8. **История заслуг** — личные транзакции и (для админов) сводка по сообществу.
9. **Профиль** — баланс, базовые настройки.

### 9.1. Проекты внутри сообщества

**Проекты остаются в MVP** — это важный функционал. Меняется только **область видимости**:

- **Есть:** вкладка «Проекты» **внутри** страницы сообщества; создание кооперативных проектов, команда проекта, тикеты/обсуждения, кошелёк проекта — в рамках этого сообщества.
- **Нет:** общая лента «все проекты платформы»; каталог проектов на уровне всего Meriter; публикация на глобальную «Биржу».

Участник видит **только проекты своей группы**. Для бизнеса: проекты — инструмент организации работы **внутри одной команды**, а не витрина на весь мир.

### 9.2. События

**Полный функционал событий** в v1:

- создание события (как публикация типа «событие»);
- RSVP (пойду / не пойду / возможно);
- приглашения участникам;
- связь с переводами заслуг в контексте события — по правилам Meriter;
- напоминания — см. варианты уведомлений в разделе 12.

### 9.3. Модерация публикаций

**Настраивается админом сообщества:**

- **Без модерации** — пост сразу в ленте и (если применимо) в чате.
- **С модерацией** — пост ждёт одобрения лида; после одобрения появляется в ленте и дублируется ботом в группу.

По умолчанию для пилота можно рекомендовать «без модерации»; группы с высоким риском спама — «с модерацией».

### 9.4. Стоимость создания контента

**Настраивается админом.** По умолчанию:

- пост — **0 заслуг**;
- опрос — по настройкам (можно 0);
- остальное — по правилам Meriter для локального сообщества.

## 10. Переход из Telegram в веб

### Внешний браузер (принятое решение)

Ссылки из бота открывают Meriter **по возможности во внешнем браузере** (Safari, Chrome), а не во встроенном окне Telegram. Так проще авторизация и стабильнее работа сложных экранов (документы, опросы).

Если Telegram не позволяет выбрать браузер — показываем **подсказку**: «Откройте ссылку во внешнем браузере» (как уже делается в текущем Meriter для email-входа).

### Автовход

Пользователь **не вводит пароль**. Авторизация через Telegram при переходе по персональной ссылке или Telegram Login.

### Что видно в вебе

Только **своё** сообщество, привязанное к группе. Нет переключателя «другие сообщества».

## 11. Роли и права

### Администраторы Telegram-группы

**Предложение:** автоматически получают роль **лида** в Meriter-сообществе. Сняли права админа в Telegram — теряют лидские права в Meriter (с grace period — **на согласовании**).

### Лид сообщества может

- менять настройки квоты, стоимости постов, модерации;
- начислять заслуги участникам (если включено в Meriter);
- управлять ролями участников;
- просматривать агрегированную историю заслуг;
- одобрять посты (если включена модерация).

### Обычный участник может

- голосовать, переводить (если разрешено), создавать контент по правилам сообщества;
- создавать проекты — если правила сообщества позволяют;
- приглашать в проект — по правилам проекта.

---

## 12. Уведомления в Telegram

**Согласовано для фазы 1 (бот): уровень «Минимум, ещё уже».**

Единственное push-уведомление в личку:

- **вам перевели заслуги**.

**Не отправляются в фазе 1:** начисления за голоса, ошибки голосования, публикация постов, события, напоминания. Публичных сообщений о переводах в группе **нет**.

---

**Уровень «Стандарт» (фаза 2 / веб-MVP)**

Минимум плюс:

- вам начислили заслуги (реакция или голос с комментарием);
- ваш пост опубликован (после модерации — если она есть);
- напоминание о событии за 24 ч и за 1 ч (если пользователь отметил «пойду»);
- приглашение на событие.

---

**Уровень «Максимум»**

Стандарт плюс:

- каждый новый пост в ленте (может быть шумно);
- активность в вашем проекте / тикете;
- изменения в совместном документе;
- ежедневное напоминание «у вас осталось N заслуг квоты».

---

**Дополнительно на согласование:**

- Можно ли **отключить** уведомления пользователем (команда боту)?
- Дублировать ли **часть** уведомлений в группу (например «Иван перевёл 5 заслуг Петру») — публично или нет?

**Для согласования:** Минимум / Стандарт / Максимум / своя комбинация.

---

## 13. Связь публикаций: веб → чат → заслуги

### Пост создан в вебе

1. Автор публикует пост (сразу или после модерации).
2. Бот отправляет в группу **карточку поста**: заголовок, excerpt, ссылка «Открыть полностью».
3. Участники ставят реакции **на сообщение бота** → заслуги начисляются автору в Meriter.
4. Рейтинг поста в вебе обновляется **синхронно**.

### Пост создан через хэштег в чате (если включён вариант B)

1. Участник пишет сообщение с `#тег`.
2. Meriter создаёт пост; сообщение участника считается «якорем» для голосов.
3. Реакции на это сообщение = голоса.

### Опрос, документ, событие, проект

- Создание — **в основном в вебе**.
- Бот может отправить в группу **короткое уведомление** со ссылкой: «Новый опрос: …», «Событие: …».
- Голосование в опросах и за варианты документов — в вебе; в чате — по желанию уведомление о результатах (если выбран уровень уведомлений).

## 14. Что НЕ входит в MVP

Явный список для согласования границ:

- Образы будущего / хаб ОБ.
- Биржа / Марафон Добра / социнвестиции.
- Tappalka (сравнение постов).
- **Общий каталог проектов** платформы (проекты **внутри** сообщества — **входят**).
- Публикация проектов на глобальную Биржу.
- Пересылка постов между сообществами.
- Создание новых сообществ пользователями (кроме сценария «новая группа = новый инстанс/onboarding» на будущее).
- Вход через email, SMS, Яндекс и др. — **только Telegram**.
- Мультиязычность — **только русский** интерфейс бота и веба MVP.
- Инвестирование заслуг в посты (если не включено отдельно в настройках проекта внутри сообщества — **уточнить**).

## 15. Масштабирование после пилота

После успешного пилота в одной группе:

- **Новая Telegram-группа** → **новое сообщество** Meriter (отдельная связка).
- Участник теоретически может состоять в **нескольких** таких группах; в веб-MVP v1 UI может показывать только одно — мульти-сообщество в UI **не обязательно** в первой версии.
- Общие глобальные ленты по-прежнему **не** требуются для этого продукта.

Это продуктовая линейка «Meriter для вашей группы», а не возврат к полному multi-hub Meriter.

## 16. Риски и ограничения

### Telegram

- API может ограничивать обработку реакций ботом — нужна проверка на этапе разработки; запасной план: голос только через reply и кнопки.
- Бот должен оставаться в группе; удаление бота **останавливает** синхронизацию (веб может работать — **на согласовании**).
- Лимиты на частоту сообщений — при активных группах нужна очередь уведомлений.

### Продуктовые

- Слишком много уведомлений → участники отключают бота в личке.
- Публичные переводы в группе могут создавать социальное давление — нужна настройка «тихий перевод».
- Модерация: при варианте «любое сообщение = пост» растёт нагрузка на лидов.

### Юридические / репутационные

- Внутренняя валюта «заслуги» — не деньги; это должно быть явно в онбординге группы.
- Дети / школьники — если аудитория пилота несовершеннолетние, согласовать с compliance отдельно.

## 18. Глоссарий

**Заслуги** — внутренняя валюта доверия внутри сообщества; не деньги, не выводятся наружу.

**Сообщество** — команда в Meriter; в MVP одно сообщество = одна Telegram-группа.

**Квота** — бесплатные «сгораемые» заслуги на день.

**Кошелёк** — накопленные заслуги, не сгорают.

**Пост** — единица контента с рейтингом; в чате представлен сообщением бота или (опционально) сообщением с хэштегом.

**Рейтинг поста** — сумма заслуг, начисленных или снятых голосами.

**Лид** — администратор сообщества в Meriter; предлагается синхрон с TG-admin.

**Проект (кооперативный)** — подкомандa внутри сообщества со своей командой, тикетами и кошельком проекта; без выхода на глобальный каталог.

**RSVP** — ответ на приглашение на событие (пойду / не пойду / возможно).

**Модерация** — одобрение поста лидом перед публикацией; вкл/выкл в настройках.

**Заморозка** — участник не может тратить/получать заслуги, пока не в группе; баланс сохраняется.

---

## 19. Чеклист приёмки фазы 1 (бот)

1. Админ добавляет бота → за ~5 минут группа с квотой и правилами **без веба** (DM-мастер).
2. Участник голосует 👍 и ❤️/🤡, переводит заслуги, смотрит `/баланс` и `/участники` с **процентами**.
3. Вышел из группы — не может тратить; вернулся — разморозка.
4. Удалил бота — сообщество заморожено, админы предупреждены в личку.
5. Получил перевод — **одно** уведомление в личку.

## 21. Фаза 2: community-web (реализовано в monorepo)

**Продукт:** отдельное Next.js-приложение `@meriter/community-web` на **`community-dobro.meriter.pro`** (prod).

**Изоляция от основного Meriter (обязательно):**
- Отдельный домен, Docker-образ, cookie `meriter_community_session`, tRPC только `/trpc/community/*` (whitelist `communityApp.*`).
- Бот и `/help` ссылаются **только** на `community-dobro.meriter.pro` — никогда на `meriter.pro/meriter/*`.
- Общий backend: `@meriter/api` + MongoDB (`MERITER_PRODUCT_MODE=telegram_mvp`).

**Маршруты:** `/login`, `/c/{communityId}/feed|projects|documents|events|merit-history|settings`, `/profile`.

**Auth:** Telegram Login Widget → `auth.authenticateTelegram` → отдельная сессия.

**Зеркало:** публикация из веба → карточка в TG-чате (`MirrorPublicationToTelegramUseCase`).

Подробнее: `12-community-web-mvp.md`, runbook: `12-community-web-runbook-ru.md`.
